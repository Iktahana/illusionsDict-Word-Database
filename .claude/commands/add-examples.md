# 例句追加スキル

`data/` ディレクトリ配下の JSON ファイルをスキャンし、例句が不足しているエントリに、**辞書として価値のある高品質な例句**を補充してください。

## 実行環境とモデル
- **ツール**: Claude Code
- **モデル**: 性能を考慮し、可能な限り上位モデル（Sonnet等）を推奨。Haiku を使用する場合は以下の制約を厳格に守ること。

## 手順

### 1. 対象の特定
- `definitions[].examples.standard` が空、または質が低い（後述の「禁止パターン」を含む）エントリを対象とする。

### 2. 例句生成の厳格ルール

各エントリに対し、**5〜8個**の例句を生成する（無理に20個作らず、質を担保すること）。

#### ✅ 推奨される例句（Good）
- **具体的シーン**: その語が実際に使われる場面（ニュース、契約書、家庭、技術現場など）を再現する。
- **コロケーション（語の繋がり）**: その語と一緒に使われやすい動詞や助詞を必ず含める。
- **定義の反映**: 例文を読むだけで、その語が「何を表しているか」が推測できる内容にする。
- **多様な語彙**: 主語や目的語に具体的な固有名詞や物質名を入れる。

#### ❌ 絶対禁止パターン（Bad/Garbage） - 以下の表現が含まれる場合は即座に破棄すること
1. **「万能テンプレート」**: 
   - 「毎日、〜は私たちの生活に欠かせません」
   - 「ビジネスシーンでは〜が重要です」
   - 「〜に関する科学的研究が進みました」
   - 「その小説では〜が物語の中心となっています」
   - 「法律では〜について明確に定義されています」
   - 「朝食の時に〜をいただきました」
2. **「メタ説明（言葉についての説明）」**:
   - 「この言葉は日常会話で頻繁に使用されます」
   - 「文脈によって意味が変わることがあります」
   - 「医学分野でも使用される専門用語です」
3. **「無意味な義務感」**:
   - 「〜を理解することが大切です」「〜について考える必要があります」

### 3. 特殊な語の扱い
- **固有名詞（校名、企業名等）**: その組織の所在地、特徴、ニュースに関連した事実に即した文にする。無理な日常会話への当てはめは禁止。
- **感嘆詞・助詞**: 実際の会話文（「」形式）として出力し、生きた語感を示すこと。

### 4. 出力フォーマット
既存の `standard` 配列を**ゴミ例句ごと上書き**してください。

```json
{
  "text": "具体的で意味のある例文",
  "source": "幻辞"
}
```

## 注意事項
- `meta.updated_at` を更新すること。
- **「例句1」「例句2」といった連番形式は絶対に禁止。**
