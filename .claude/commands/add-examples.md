# 例句追加スキル

`data/` ディレクトリ配下の全 JSON ファイルをスキャンし、例句が不足している辞書エントリを特定して、各エントリに **20個の例句** を追加してください。

## 手順

### 1. 対象エントリの検索

`data/` 以下の全 `.json` ファイルを走査し、以下の条件に該当するエントリを抽出する：

- `definitions[].examples` が存在しない、または空オブジェクト `{}`
- `definitions[].examples.standard` が存在しない、空配列 `[]`、または要素数が 20 未満

始める前に、例文をいくつ追加する必要があるか教えてください。

### 2. 例句の生成

各対象エントリについて、以下のルールに従い **20個** の例句を生成する：

- **最低5つの異なる領域**（ドメイン）から例句を作成すること。領域の例：
  - 日常会話・生活
  - ビジネス・経済
  - 科学・技術
  - 文学・芸術
  - 教育・学問
  - 医療・健康
  - 法律・政治
  - スポーツ・娯楽
  - 自然・環境
  - 歴史・文化
  - 料理・食文化
  - 旅行・交通

- 例句は自然な日本語で、その語の典型的な用法を示すものにする
- 語の意味・ニュアンスが文脈から理解できる文を作る
- 同じパターンの繰り返しを避け、多様な構文・文体を用いる
- 1文は 20〜80 文字程度を目安とする

### 3. 出力フォーマット

`definitions[].examples.standard` 配列に以下の形式で追加する：

```json
{
  "text": "例句のテキスト",
  "source": "幻辞"
}
```

既存の `examples.standard` に要素がある場合は、既存の要素を保持したまま、合計 20 個になるよう不足分を追加する。`examples.literary` には触れないこと。

### 4. ファイル書き込み

- 変更した JSON ファイルのみを上書き保存する
- `json.dump` 相当で `ensure_ascii=False, indent=2` のフォーマットを維持する
- `meta.updated_at` を現在の UTC 時刻（ISO 8601 形式）に更新する

### 5. 進捗報告

処理の進捗をユーザーに報告する：
- 対象エントリ総数
- 処理済み件数（10件ごと、または完了時）
- エラーが発生した場合はファイルパスとエラー内容

## 注意事項

- 一度に大量のファイルを処理する場合は、ディレクトリ単位（あ行、か行など）でバッチ処理する
- 既存データ（`entry`, `reading`, `grammar`, `relations`, `meta` 等）は一切変更しない
- `examples.literary` の既存データも保持する
